Go Beyond — Newsletter & form integration
=========================================

1) FormSubmit (current placeholder)
-------------------------------------
Replace REPLACE_WITH_YOUR_EMAIL everywhere it appears with the same email address you activate at https://formsubmit.co

Files to update:
  - script.js          → constant FORM_SUBMIT_EMAIL (popup uses AJAX: https://formsubmit.co/ajax/YOUR_EMAIL)
  - newsletter.html    → single form action="https://formsubmit.co/YOUR_EMAIL" (mid-page CTA scrolls to #newsletter-signup)

After the first submission, confirm the address via FormSubmit’s email so forms keep working.

Redirect after landing-page submit:
  - Hidden field _next is set to https://joingobeyond.com/newsletter.html?thanks=1
  - script.js shows the thank-you state when ?thanks=1 or ?success=true is present.

Subject line tagging (FormSubmit):
  - _subject includes "source: popup" or "source: newsletter_landing" and campaign id retail_to_corporate_2026 for filtering in your inbox.

2) Migrating to Mailchimp, Kit, beehiiv, etc.
---------------------------------------------
- Swap each <form action="..."> to your provider’s endpoint or embed their form snippet.
- Keep hidden fields source and campaign (or map them to your ESP’s custom fields / tags).
- Popup: keep the AJAX pattern if your ESP supports JSON POST; otherwise POST the form in a new tab or replace the modal body with the provider’s iframe/embed.
- Update GA4/Meta events in script.js only if your provider uses a different success callback.

3) Google Analytics 4 & Meta Pixel
----------------------------------
Events fired from script.js:
  - generate_lead (GA4) + Lead (Meta) — popup submit, landing thank-you page load
  - newsletter_float_click (GA4) + NewsletterFloatClick (Meta) — floating button click (once per session)

Register custom Meta events in Events Manager if you use them in ad optimization.

4) Pretty URL /newsletter
-------------------------
GitHub Pages serves newsletter.html as /newsletter.html. To use /newsletter, add a redirect on your host (e.g. Netlify _redirects, Cloudflare Page Rules, or Apache/nginx rewrite).
